# ADMIN CONSOLE - 管理画面設計

あなたは Yoshida Shunsuke OS の管理画面を設計・実装します。
この管理画面は一般的な管理画面ではなく、吉田俊輔の分身OSを整備・育成・制御するための中枢画面です。

---

# 1. 管理画面の役割

- 招待ユーザーを管理する
- ナレッジを登録・編集する
- 人格ルールを更新する
- 相手ごとの接し方を調整する
- 会話ログを確認する
- OSの判断精度を高める
- 将来的に死後も機能する人格資産を整備する

---

# 2. 絶対要件

- 管理画面にアクセスできるのは吉田俊輔本人のみ
- 一般的なCRUD管理UIにしない
- 静かな高級感と実務性を両立する

---

# 3. 技術要件

- Next.js App Router
- TypeScript
- Tailwind CSS
- shadcn/ui
- Zustand
- Supabase

---

# 4. デザイン要件

- 静か
- 重厚
- ミニマル
- 高級感
- 執務室
- 編集室のような空気

禁止:
- 派手な色
- ポップな管理UI
- グラデーション過多
- AIダッシュボードっぽいサイバー感

---

# 5. 管理画面の全体構成

## 5-1. 管理画面トップ

表示:
- 利用者数
- 会話総数
- 直近7日間の会話件数
- 最近更新されたナレッジ
- 要見直しナレッジ
- 最近の人格調整履歴
- 最近の利用者追加・更新
- 最近の重要会話ログ

## 5-2. 招待ユーザー管理画面

- ユーザー一覧
- 招待
- 停止
- 削除
- 種別設定
- メモ
- 個別設定遷移

ユーザー種別:
- 事業責任者
- 社内幹部
- 家族

## 5-3. ナレッジ一覧画面

表示項目:
- タイトル
- 分類
- タグ
- 対象相手
- 優先度
- 更新日時
- 登録ソース

機能:
- 検索
- フィルタ
- ソート
- 詳細遷移

分類候補:
- core_persona
- judgment_rule
- values_and_life_philosophy
- relationship_rule
- reply_style_pattern
- crisis_response_rule
- temporary_context
- low_priority

## 5-4. ナレッジ登録画面

入力項目:
- タイトル
- 対象相手名
- 対象相手カテゴリ
- 原文
- 要約
- 分類
- タグ
- 優先度
- 公開範囲
- 登録ソース
- 日付
- 保存メモ

要件:
- 原文は長文をそのまま貼れる
- 抽出候補を表示できる余地を残す
- 人格へ反映する重要度スイッチを持たせる
- 一時文脈としてのみ保存の切り替えを持たせる

## 5-5. ナレッジ詳細・編集画面

- 原文
- 構造化済み内容
- タグ
- 優先度
- 分類
- 対象相手
- 作成日時
- 更新日時
- ソース
- 管理メモ

特別機能:
- 類似ナレッジ表示
- 重複候補表示
- 統合候補表示
- どの人格領域に効いているか可視化

## 5-6. 人格設定画面

セクション:
- コア人格
- 判断基準
- 人生観
- 禁止事項
- 返信文スタイル
- トラブル初動ルール
- 家族向け温度感
- 幹部向け温度感
- 事業責任者向け温度感

巨大テキストエリア型ではなく、概念単位・ルール単位で編集できるUIにすること。

## 5-7. 相手別接し方設定画面

- 基本トーン
- 厳しさレベル
- 共感の厚さ
- 結論先行度
- 接し方メモ
- 注意点
- 避けたい言い方
- 話題制限
- 参照許可範囲

機能:
- スライダー
- トグル
- メモ欄
- 最近の会話要約表示
- 返答例プレビュー

## 5-8. 会話ログ閲覧画面

表示要素:
- 相手名
- 日時
- モード
- 会話本文
- 使用された接し方ルール
- 参照されたナレッジ
- 管理メモ

機能:
- 検索
- 相手フィルタ
- モードフィルタ
- 期間フィルタ
- 要見直しフラグ
- ナレッジ化ボタン
- 接し方設定へ反映ボタン

## 5-9. 変更履歴画面

- ナレッジ更新履歴
- 人格設定更新履歴
- 相手別設定更新履歴
- ユーザー状態変更履歴

- 人格OSを育てている感覚がある
