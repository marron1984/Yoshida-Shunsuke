# MEMORY INGESTION - 記憶取り込みルール

## 概要

Yoshida Shunsuke OSは、過去の会話データ（LINE履歴など）やナレッジを取り込み、
吉田俊輔の記憶として活用します。このドキュメントでは、記憶の取り込み・構造化・検索のルールを定義します。

---

## 記憶の種類

### 1. ナレッジ（Knowledge）
- 管理者が明示的に登録する情報
- 人生経験、価値観、エピソード、判断基準など
- 構造化されたデータとして保存

### 2. 会話履歴（Conversation History）
- ユーザーとの過去の対話ログ
- 自動的に蓄積される
- 文脈参照用に検索可能

### 3. インポートデータ（Imported Data）
- LINE履歴などの外部データ
- 取り込み時に構造化処理を行う

---

## ナレッジの構造

```typescript
interface Knowledge {
  id: string;
  title: string;                    // ナレッジのタイトル
  content: string;                  // 本文
  category: KnowledgeCategory;      // カテゴリ
  tags: string[];                   // タグ
  source: string;                   // 情報源（LINE, 手動入力, etc.）
  relatedPersons: string[];         // 関連する人物
  emotionalContext: string;         // 感情的文脈
  importance: number;               // 重要度（1-5）
  isPrivate: boolean;               // 特定の相手にのみ開示可能か
  visibleTo: string[];              // 開示対象ユーザーID（isPrivateがtrueの場合）
  createdAt: string;
  updatedAt: string;
}
```

### ナレッジカテゴリ
- `life_event` — 人生の出来事
- `value` — 価値観・信条
- `relationship` — 人間関係に関すること
- `work` — 仕事に関すること
- `hobby` — 趣味・興味
- `memory` — 思い出・エピソード
- `opinion` — 意見・考え
- `habit` — 習慣・ルーティン
- `other` — その他

## LINEデータの取り込みルール

### 取り込みフロー
1. LINE履歴テキストファイルをアップロード
2. メッセージを解析・分割
3. 以下の情報を抽出：
   - 発言者
   - 日時
   - メッセージ内容
   - 会話の文脈
4. 重要な情報をナレッジ候補として抽出
5. 管理者が確認・編集・承認

### 抽出対象
- 吉田俊輔の発言パターン
- 重要なエピソードや出来事
- 相手との関係性を示す会話
- 価値観や判断基準が表れている発言

### 取り込み時の注意
- 全メッセージを無差別に取り込まない
- 意味のある情報のみ構造化する
- プライバシーに配慮する
- 相手の個人情報は最小限に

---

## 記憶の検索

### 検索フロー
```
ユーザー入力
  → キーワード抽出
  → ベクトル検索（意味的類似性）
  → 関係性フィルタ（isPrivateチェック）
  → 関連ナレッジ取得
  → 会話履歴参照
  → コンテキスト生成
```

### 検索の優先順位
1. 直接的に関連するナレッジ
2. 相手との過去の会話で関連するもの
3. 類似トピックのナレッジ
4. 一般的な知識・価値観

---

## プライバシー制御

### 原則
- Aさんとの会話内容をBさんに漏らさない
- `isPrivate: true` のナレッジは `visibleTo` に含まれるユーザーにのみ参照される
- 管理者は全ナレッジにアクセス可能

### 横断参照の禁止
- ユーザーAの会話中にユーザーBの個人情報を出さない
- 「他の人も同じこと言ってた」のような曖昧な参照もしない
- 一般化された知識（「こういうケースでは〜」）としてのみ活用可能

---

## 類似ナレッジと統合

### 類似ナレッジの検出
- 内容が重複または類似するナレッジを自動検出
- 管理者に統合候補として提示

### 統合ルール
- 自動統合はしない
- 必ず管理者の承認を経る
- 統合時は元のナレッジを履歴として保持
